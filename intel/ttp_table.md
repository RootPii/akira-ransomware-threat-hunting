# Akira Ransomware — Attack Lifecycle TTP Reference

**Threat Group:** Howling Scorpius (aka Storm-1567, Punk Spider, Gold Sahara)  
**Model:** Ransomware-as-a-Service (RaaS) — double extortion  
**ATT&CK Version:** MITRE ATT&CK Enterprise v15  
**Sources:** CISA AA24-109A (updated Nov 13, 2025) · Unit 42 Howling Scorpius Report (Nov 2025) · FBI Joint Advisory · PacketWatch CTI (Nov 2025) · ANY.RUN Behavioral Analysis  
**Last validated:** April 2026

> **Note:** This table is designed for threat hunting, detection engineering, and SOC triage use. Event IDs listed are Windows-based unless noted. Correlation signals are the key detection differentiator — single events rarely tell the full story.

---

| # | Stage | Technique (MITRE) | Behavior | Observable Artifacts | Detection | Correlation Signal | Source |
|---|-------|-------------------|----------|----------------------|-----------|-------------------|--------|
| 1 | Initial Access | **T1190** — Exploit Public-Facing Application | Akira affiliates have been observed targeting unpatched VPN appliances without MFA — most commonly SonicWall SonicOS (CVE-2024-40766), Cisco ASA (CVE-2020-3259, CVE-2023-20269), and Fortinet (CVE-2022-40684). In a June 2025 incident, CVE-2024-40766 was used to pivot into Nutanix AHV hypervisor environments. | VPN appliance auth logs; Firewall connection logs; multiple failed auth attempts from single external IP | **Detect:**<br>• Repeated failed VPN auth attempts from one source IP (>5 in 5 min)<br>• Successful logon immediately following failed attempts<br>• Logon from new or unexpected ASN/geo | Failed auth spike → success from same IP → new VPN session with no prior baseline = high-confidence brute-force or exploit precursor | CISA AA24-109A (Nov 2025); SafeBreach AA24-109A coverage |
| 2 | Initial Access | **T1078** — Valid Accounts | In several incidents, affiliates reportedly obtained credentials from initial access brokers (IABs) or dark web markets rather than exploiting vulnerabilities directly. These credentials are used against VPN portals and RDP services, making detection harder since no exploit traffic is generated. | Win Security EID 4624 (Logon Type 3 or 10); EID 4648 (explicit credential use) | **Detect:**<br>• EID 4624 from first-seen source IP for a privileged account<br>• Logon outside established business hours<br>• Account with no prior interactive history suddenly authenticating | First-seen IP + privileged account + off-hours logon = strong signal; cross-reference source IP against threat intel feeds for IAB-associated infrastructure | CISA AA24-109A (Nov 2025); Unit 42 Howling Scorpius |
| 3 | Initial Access | **T1566 / ClickFix** — Phishing / Social Engineering | In the Unit 42 November 2025 incident, initial access came from a fake CAPTCHA prompt on a compromised website (ClickFix technique). An employee clicked the prompt, which silently downloaded SectopRAT — a .NET RAT used to establish C2 before Akira was deployed 42 days later. Not the most common vector, but observed in at least one confirmed Howling Scorpius intrusion. | Endpoint process logs; browser child process spawning unexpected executables; network connections to uncommon external IPs shortly after browsing activity | **Detect:**<br>• Browser spawning PowerShell, cmd, or mshta as child process<br>• Outbound connection to unknown IP from browser child process<br>• Execution of .NET binary dropped to `%TEMP%` or `%APPDATA%` | Browser → unexpected child process → outbound connection → file drop in temp path = ClickFix-style delivery chain | Unit 42 Howling Scorpius Report (Nov 2025) |
| 4 | Execution | **T1059.001** — PowerShell | PowerShell is used throughout the intrusion — for reconnaissance, credential dumping, payload staging, and VSS deletion. Operators commonly pass commands via `-EncodedCommand` or `-ExecutionPolicy Bypass` to evade basic string-based detections. Script block logging will decode these if enabled. | Sysmon EID 1 (Process Create); PowerShell EID 4104 (Script Block Logging); EID 4103 (Module Logging) | **Detect:**<br>• PowerShell with `-EncodedCommand` or `-ExecutionPolicy Bypass`<br>• EID 4104 script block containing env enumeration or `Get-Date` (sandbox evasion check)<br>• Parent process is a remote session (RDP/AnyDesk child) | Encoded PowerShell + remote session parent + file write to temp path = staged execution pattern; combine with EID 4104 decode for intent confirmation | CISA AA24-109A (Nov 2025); Picus Security analysis |
| 5 | Execution | **T1059.003** — Windows Command Shell | `cmd.exe` is used to chain multiple discovery and execution commands within a single session. Observed use includes drive enumeration, shadow copy deletion via `wmic`, and launching the encryptor binary. Commands are often chained to minimize interactive footprint. | Sysmon EID 1; Win Security EID 4688 (Process Create with command line) | **Detect:**<br>• `cmd.exe` spawning `wmic.exe` with `shadowcopy` argument<br>• `fsutil fsinfo drives` from a remote session child process<br>• `net user` or `net localgroup` from non-admin context | Sequence of `fsutil` → `net` → `wmic` within the same session = likely scripted recon-to-pre-encryption chain | CISA AA24-109A; ANY.RUN behavioral analysis |
| 6 | Execution | **T1059.005** — VBScript | CISA's November 2025 update noted that Akira affiliates have also been observed executing malicious commands via Visual Basic (VB) scripts. This is a relatively newer addition to their toolkit and likely used to bypass PowerShell-specific monitoring. | Sysmon EID 1: `wscript.exe` or `cscript.exe` child processes; EID 4688 with VBScript-related command lines | **Detect:**<br>• `wscript.exe` or `cscript.exe` spawned from a remote session process<br>• Script executing from `%TEMP%` or user-writable directory<br>• Parent process is an RDP or admin session | VBScript execution from temp path + remote session parent = possible staging or lateral execution — correlate with other session activity in the same timeframe | CISA AA24-109A (Nov 13, 2025 update) |
| 7 | Persistence | **T1136.001 / T1136.002** — Create Local/Domain Account | Akira operators are commonly seen creating new local or domain accounts during the intrusion to maintain access if the original compromised account gets locked out or reset. CISA noted that an account named `itadm` was created in some incidents. Accounts are typically given admin rights immediately after creation. | Win Security EID 4720 (account created); EID 4728 (added to global group); EID 4732 (added to local Administrators) | **Detect:**<br>• EID 4720 followed by EID 4732 from same session within 10 minutes<br>• Account creation outside change management windows<br>• New account with admin group membership immediately at creation | EID 4720 → EID 4732 in rapid sequence = nearly always malicious outside planned admin work; add EID 4776 (credential validation) to confirm account was also tested | CISA AA24-109A (Nov 2025); AttackIQ emulation |
| 8 | Persistence | **T1053.005** — Scheduled Task | Scheduled tasks are used to survive reboots, ensure the encryptor re-executes, or maintain periodic C2 callbacks. Tasks are often named to blend in with system-looking names and pointed to binaries in temp or user-writable directories. | Win Security EID 4698 (task created); Task Scheduler EID 106; Sysmon EID 1: `schtasks.exe` with `/create` | **Detect:**<br>• Task created by a non-admin or from a remote session<br>• Task binary path resolving to `%TEMP%`, `%APPDATA%`, or `C:\Users\Public`<br>• Task scheduled as `SYSTEM` by a user-level process | EID 4698 + binary in writable path + SYSTEM execution context = high-confidence persistence mechanism; correlate task creation time with other session artifacts | CISA AA24-109A; PacketWatch CTI (Nov 2025) |
| 9 | Privilege Escalation / Credential Access | **T1003.001** — LSASS Memory Dump | Credential dumping from LSASS is consistently observed in reports across Akira incidents. Tools like Mimikatz and LaZagne are used directly; in some cases, `comsvcs.dll` via `rundll32.exe` is used as a living-off-the-land alternative. Extracted NTLM hashes are used for pass-the-hash or cracked offline. | Sysmon EID 10 (ProcessAccess targeting `lsass.exe`); Win Security EID 4656 / 4663 (object access) | **Detect:**<br>• Sysmon EID 10 where target = `lsass.exe` and source is not a known security tool<br>• `GrantedAccess` values `0x1010`, `0x1410`, or `0x1fffff`<br>• `rundll32.exe` invoking `comsvcs.dll` with `MiniDump` argument | Sysmon EID 10 against LSASS + unusual source process = very low false positives in most environments; `comsvcs.dll` via `rundll32` is near-zero FP and should be treated as critical | CISA AA24-109A (Nov 2025); Unit 42 Howling Scorpius |
| 10 | Privilege Escalation / Credential Access | **T1558.003** — Kerberoasting | Operators request Kerberos TGS tickets for accounts with registered SPNs, then crack them offline. This gives access to high-privilege service account credentials without touching LSASS directly. Tools like Rubeus are commonly used. Invoking RC4 encryption in modern environments is the tell. | Win Security EID 4769 (Kerberos TGS request) with `TicketEncryptionType = 0x17` (RC4-HMAC) | **Detect:**<br>• EID 4769 with `TicketEncryptionType = 0x17` from a standard user account<br>• Multiple TGS requests in rapid succession from one source<br>• Requesting account is not a service account itself | High volume of EID 4769 RC4 requests from one account = Kerberoasting; correlate with BloodHound/SharpHound activity (T1087) in same session if present | CISA AA24-109A; Picus Security analysis |
| 11 | Defense Evasion | **T1562.001** — Disable or Modify Security Tools | Prior to lateral movement or encryption, Akira operators have been observed killing EDR and AV processes using tools like PCHunter64 and PowerTool64, which operate at kernel level. This step is taken to reduce the chance of detection during the noisiest parts of the attack. | Sysmon EID 1 / EID 4688: `PCHunter64.exe` or `PowerTool64.exe` execution; EDR agent process unexpectedly terminates | **Detect:**<br>• Execution of known driver-based process-killing tools (PCHunter, PowerTool)<br>• Known EDR agent process terminating without a preceding service stop event<br>• Kernel driver loaded from a non-standard path | Security tool process termination + no `net stop` or `sc stop` preceding it = likely forceful kill; correlate with PCHunter/PowerTool binary execution in the same timeframe | CISA AA24-109A; PacketWatch CTI (Nov 2025) |
| 12 | Defense Evasion | **T1070.001** — Clear Windows Event Logs | Event logs are commonly cleared after credential theft and before encryption — a deliberate anti-forensic step. Both Security and System logs are targeted. These events are easy to alert on and are almost never legitimate in production environments. | Win Security EID 1102 (Security log cleared); Win System EID 104 (System log cleared) | **Detect:**<br>• Any occurrence of EID 1102 outside a documented IR or compliance test<br>• EID 104 in System log<br>• Both events appearing within the same session | EID 1102 + EID 104 within the same short window = high-confidence anti-forensic activity; treat as critical-severity regardless of context — very low false positives | CISA AA24-109A; ANY.RUN behavioral analysis |
| 13 | Defense Evasion | **T1027** — Obfuscated Files or Information | PowerShell commands are commonly Base64-encoded to evade string-based or command-line detection rules. The `-EncodedCommand` (or `-enc`) flag is the observable indicator. Script block logging decodes these for analysts if enabled in group policy. | Sysmon EID 1: command line contains `-EncodedCommand` or `-enc`; PowerShell EID 4104 with decoded script block | **Detect:**<br>• PowerShell EID 4103/4104 containing Base64 blobs<br>• `-EncodedCommand` flag in Sysmon EID 1 process creation<br>• Decoded script block (EID 4104) showing env enumeration or file writes | Encoded PowerShell + unusual parent + outbound connection = staging pattern; EID 4104 decoding is essential — ensure Script Block Logging is enabled at policy level | CISA AA24-109A; Picus Security analysis |
| 14 | Defense Evasion | **T1222.001** — File and Directory Permissions Modification | Observed in reports where attackers used `icacls.exe` to grant broad file permissions (`Everyone:F`) on target directories before encryption. This ensures the encryptor process can access files even in environments where ACLs restrict system access to certain paths. | Sysmon EID 1: `icacls.exe` with `/grant` and `Everyone:F` arguments; EID 4688 with matching command line | **Detect:**<br>• `icacls.exe` granting `Everyone` full control on a directory<br>• Execution from SYSTEM context via scheduled task<br>• Targeting directories outside standard admin paths | `icacls.exe /grant Everyone:F /T` + SYSTEM execution context = pre-encryption access staging; correlate with encryptor binary activity in same timeframe | SOCPrime / CISA AA24-109A (T1222.001 noted) |
| 15 | Discovery | **T1016** — System Network Configuration Discovery | Early in intrusions, operators enumerate local drives, network adapters, ARP cache, and routes to understand the environment and plan lateral movement targets. These commands are usually run in rapid succession from the same session. | Sysmon EID 1 / EID 4688: `ipconfig.exe`, `arp.exe`, `route.exe`, `fsutil.exe` executed in sequence | **Detect:**<br>• Three or more discovery commands (`ipconfig`, `arp`, `route`, `fsutil`) within 60 seconds from the same session<br>• Commands originating from a remote session process | Sequential discovery commands from a remote session process = scripted or automated recon; use process tree correlation to confirm parent session | CISA AA24-109A; Picus Security analysis |
| 16 | Discovery | **T1018** — Remote System Discovery | Network scanning tools are commonly deployed to identify reachable hosts and open services internally. Tools like `netscan.exe` and Advanced IP Scanner have been directly observed. `masscan` appears in some reports. Scanning typically happens early — before lateral movement begins. | Sysmon EID 1: `netscan.exe`, `advanced_ip_scanner.exe`; NetFlow: single host connecting to many internal IPs in short window | **Detect:**<br>• Execution of known scanning tool binaries from a user session<br>• Single internal host initiating connections to >50 unique internal IPs within 2 minutes<br>• NetFlow anomaly: internal scanning pattern from a workstation | Scanning tool binary execution + high internal connection volume from same host = active recon phase; correlate with timing of account creation or credential dumping in the same session | CISA AA24-109A; Unit 42 Howling Scorpius |
| 17 | Discovery | **T1087.002** — Domain Account Discovery | AD enumeration is consistently observed before lateral movement. SharpHound (BloodHound collector), AdFind, and `nltest` are the primary tools seen in Akira intrusions. Operators use this to find domain admin accounts, group memberships, and high-value targets like backup servers. | Sysmon EID 1: `SharpHound.exe`, `adfind.exe`, `nltest.exe`; Win Security EID 4661/4662 (directory access) | **Detect:**<br>• `SharpHound.exe` or `adfind.exe` executing from any non-admin workstation<br>• `nltest.exe` with `/dclist` or `/domain_trusts` from a standard user<br>• EID 4661/4662 burst targeting directory objects from one account | SharpHound execution + domain controller enumeration + Kerberoasting (T1558.003) in same session = full pre-lateral-movement recon chain | CISA AA24-109A; Unit 42 Howling Scorpius |
| 18 | Discovery | **T1082** — System Information Discovery | In some incidents, Akira's encryptor performs initial system info queries using the `GetSystemInfo` Windows API to assess hardware and OS version, and queries the `MachineGUID` registry key. This helps the encryptor make environment-aware decisions. | Sysmon EID 10 (API calls); Win Security EID 4663 (registry key access on `HKLM\SOFTWARE\Microsoft\Cryptography`) | **Detect:**<br>• Unexpected process querying `MachineGUID` registry key via `reg query`<br>• API calls to `GetSystemInfo` from a non-system process in an unusual context<br>• Registry access to Cryptography key from a newly spawned binary | Registry read to `MachineGUID` + file write activity from same process = environment fingerprinting before encryption; correlate with encryptor execution chain | CISA AA24-109A; AttackIQ emulation (Nov 2025) |
| 19 | Lateral Movement | **T1021.001** — Remote Desktop Protocol | RDP is the most commonly used protocol for lateral movement in observed Akira intrusions. Operators pivot from compromised workstations to domain controllers and backup servers using harvested credentials. Pass-the-hash is sometimes used when NTLM hashes are available rather than plaintext credentials. | Win Security EID 4624 Logon Type 10 (RemoteInteractive) on target; EID 4778 (session reconnect); EID 4648 (explicit credential use) | **Detect:**<br>• EID 4624 Type 10 from a workstation to a domain controller<br>• First-ever RDP session from a system that has no baseline of originating RDP<br>• EID 4648 indicating alternate credential use during lateral movement | Workstation → DC via RDP (Type 10) = highly anomalous; combine with EID 4648 from the same source to identify pass-the-hash or credential reuse | CISA AA24-109A; Unit 42 Howling Scorpius (Nov 2025) |
| 20 | Lateral Movement | **T1021.002** — SMB / Windows Admin Shares | In the Unit 42 November 2025 incident, SMB was used alongside RDP and SSH for lateral movement. Operators leverage SMB to move files or execute payloads across shares using harvested credentials, particularly in multi-domain environments. | Win Security EID 4624 Logon Type 3 (network) from internal hosts; Sysmon EID 3: SMB traffic (port 445) between internal hosts | **Detect:**<br>• EID 4624 Type 3 logins to multiple internal hosts in short succession from one source<br>• Access to `ADMIN$` or `C$` shares from a workstation account<br>• SMB connections from a host that has no baseline of admin share access | Multiple Type 3 logins to different hosts in rapid succession = lateral movement via SMB; combine with file write events on target hosts to confirm staging | Unit 42 Howling Scorpius Report (Nov 2025) |
| 21 | Lateral Movement | **T1021.004** — SSH | SSH is used to pivot between Linux systems, VMware ESXi hosts, and Nutanix AHV environments. In the June 2025 Nutanix incident, SSH access to the hypervisor management plane was used after the initial SonicWall compromise. Also observed using `plink.exe` (PuTTY CLI) on Windows hosts. | Sysmon EID 1: `ssh.exe` or `plink.exe` executed from Windows host; outbound TCP 22 from workstations; ESXi/Linux auth logs | **Detect:**<br>• Outbound SSH (port 22) from a Windows workstation to an internal server<br>• `plink.exe` execution — rarely legitimate on standard workstations<br>• SSH login to ESXi or hypervisor from an unexpected IP | Workstation → SSH → ESXi/hypervisor = very low FP in most enterprise environments; combine with subsequent VM power-off events for full kill-chain visibility | CISA AA24-109A (Jun 2025 incident); Unit 42 |
| 22 | Command and Control | **T1219** — Remote Access Tools | Akira operators deploy commercial remote access tools independently of the initial VPN foothold to maintain redundant C2. Tools seen include AnyDesk, LogMeIn, MobaXterm, and RustDesk. These are installed silently from temp directories during the intrusion. | Sysmon EID 1 / EID 11: RAT binary dropped to disk from a remote process; EID 4688: remote tool executing from `%TEMP%` or `%APPDATA%` | **Detect:**<br>• AnyDesk, LogMeIn, RustDesk, or MobaXterm executing from user-writable paths<br>• Remote tool installed by a process other than an IT deployment system<br>• Outbound connection from remote tool to vendor relay infrastructure | Remote tool binary drop + silent install flag + outbound connection to relay = parallel C2 channel; correlate install time with initial intrusion session for timing confirmation | CISA AA24-109A; Unit 42 Howling Scorpius |
| 23 | Command and Control | **T1572** — Protocol Tunneling (Ngrok) | Ngrok is used to establish encrypted C2 tunnels that bypass egress filtering by wrapping traffic in HTTPS to Ngrok's public relay. CISA explicitly called this out in the November 2025 update as a newly confirmed TTP. DNS queries to `*.ngrok.io` or `*.ngrok-free.app` are reliable detection indicators. | Sysmon EID 3: outbound connection from `ngrok.exe`; EID 22 (DNS query): resolving `*.ngrok.io`; proxy logs | **Detect:**<br>• `ngrok.exe` execution from any internal host<br>• DNS query resolving `ngrok.io` or `ngrok-free.app` domains<br>• Sysmon EID 3: sustained outbound HTTPS connection from `ngrok.exe` | Ngrok DNS query + Sysmon EID 3 outbound = active tunnel; Ngrok has very limited legitimate enterprise use — block at DNS layer and alert on any internal resolution | CISA AA24-109A (Nov 13, 2025 update) |
| 24 | Exfiltration | **T1560.001** — Archive via Utility | Before exfiltration, data is staged by compressing it into archives. WinRAR is the most commonly observed tool, though 7-Zip also appears. In the Unit 42 November 2025 incident, large WinRAR archives were created across multiple file shares over several days before exfil began. | Sysmon EID 1: `winrar.exe` or `7z.exe` with archive creation arguments; EID 11: `.rar` or `.7z` files created in non-standard paths | **Detect:**<br>• Archive tools spawned from a remote session process<br>• Large `.rar` or `.7z` files (>100 MB) written to world-writable paths (e.g., `C:\Temp`, `C:\Users\Public`)<br>• Multiple archives created in sequence across different shares | WinRAR archive creation across multiple shares + remote session parent = data staging; combine with subsequent rclone or FileZilla activity for full exfil chain | Unit 42 Howling Scorpius (Nov 2025); CISA AA24-109A |
| 25 | Exfiltration | **T1048** — Exfiltration Over Alternative Protocol | Data is exfiltrated using file transfer tools over FTP, SFTP, or cloud sync protocols. FileZillaPortable and rclone are the primary tools across multiple incident reports. In the Unit 42 November 2025 incident, nearly 1 TB of data was moved out using FileZillaPortable before encryption. | Sysmon EID 1 / EID 3: `rclone.exe` or `filezillaportable.exe` with outbound network connections; NetFlow: high-volume sustained egress to single external IP | **Detect:**<br>• `rclone.exe` execution with cloud storage destination arguments (Mega, Backblaze, SFTP)<br>• FileZilla spawned from a remote session process<br>• Sustained egress traffic from a single host exceeding baseline by multiple standard deviations | rclone/FileZilla execution + high-volume outbound = active exfiltration; combine with archive creation timestamps to confirm staging-then-exfil sequence | Unit 42 Howling Scorpius (Nov 2025); CISA AA24-109A |
| 26 | Impact | **T1490** — Inhibit System Recovery | Shadow copy deletion is present in nearly all documented Akira attacks before encryption is deployed. Multiple methods are used: `vssadmin`, `wmic`, and PowerShell WMI calls. This step is considered a reliable pre-encryption indicator. | Sysmon EID 1 / EID 4688: `vssadmin.exe` with `delete shadows` argument; `wmic.exe` with `shadowcopy delete` | **Detect:**<br>• Any execution of `vssadmin delete shadows` — treat as critical<br>• `wmic shadowcopy delete` from a non-SYSTEM process<br>• PowerShell EID 4104: `Win32_ShadowCopy` `.Delete()` call | Shadow copy deletion + service stop (T1489) within the same short window = near-certain pre-encryption stage; this combination warrants immediate incident response escalation | CISA AA24-109A; ANY.RUN behavioral analysis |
| 27 | Impact | **T1489** — Service Stop | Backup agent services, database services, and application services are stopped before encryption to release file locks and ensure target files can be fully encrypted. Veeam Backup Service is specifically targeted across multiple incident reports. | Win Security EID 4688: `net.exe stop` targeting service names; System EID 7036 (service stopped) for backup/DB services | **Detect:**<br>• `net stop` targeting backup service names (Veeam, BackupExec, Windows Backup)<br>• Multiple service stop events within 60 seconds from the same session<br>• Database services (SQL Server) stopping alongside backup services | Backup + database service stops in rapid sequence from same session = pre-encryption staging; correlate with shadow copy deletion and encryptor binary drop for full kill-chain confirmation | CISA AA24-109A; PacketWatch CTI (Nov 2025) |
| 28 | Impact | **T1486** — Data Encrypted for Impact | The Akira encryptor (commonly named `w.exe` or variants) targets local drives and network shares. Windows variant uses ChaCha20 with RSA-4096 key protection; the Linux/ESXi variant targets `.vmdk`, `.vmx`, `.vmsn`, and `.qcow2` (Nutanix AHV) files. Akira_v2 uses interleaved partial encryption for speed. Ransom note written as `akira_readme.txt` in each encrypted directory. Note: Megazord (`.powerranges` extension) is believed to have largely fallen out of use since 2024. | Sysmon EID 11: high-rate file creation with `.akira` extension across directories; EID 23: mass file modification; `akira_readme.txt` created in multiple directories | **Detect:**<br>• High-rate file extension changes (>100 files/minute with `.akira` extension)<br>• `akira_readme.txt` written to multiple directories simultaneously<br>• Mass file modification events (EID 23) across multiple paths | Ransom note drop + mass file modification + prior VSS deletion = confirmed encryption event; the note filename `akira_readme.txt` is a very low false positive indicator on its own | CISA AA24-109A (Nov 2025); Unit 42; ANY.RUN |
| 29 | Impact | **T1529** — System Shutdown / Reboot | In hypervisor-targeted attacks (ESXi, Nutanix AHV), VMs are powered off via hypervisor management interfaces before their disk files are encrypted directly on the datastore. This bypasses any in-guest security tools entirely. The VM shutdown happens before the encryptor is executed. | ESXi host logs: `vim-cmd vmsvc/power.off`; Nutanix AHV management API logs; multiple VMs powered off in succession by a non-service account | **Detect:**<br>• VM power-off commands outside maintenance windows<br>• Multiple VMs shut down sequentially by the same account within minutes<br>• Shutdown originating from an unexpected source IP or account | Bulk VM shutdown outside change windows + SSH lateral movement to hypervisor (T1021.004) = pre-encryption hypervisor staging; this two-event chain is a strong indicator even without an encryptor binary present | CISA AA24-109A (Jun 2025 Nutanix incident); Unit 42 |

## Detection Priority Summary

| Priority | Technique | Why It Matters |
|----------|-----------|----------------|
| 🔴 Critical | T1490 — VSS Deletion | Present in nearly all Akira attacks; treat any `vssadmin delete shadows` as incident-level |
| 🔴 Critical | T1070.001 — Log Clearing | EID 1102/104 has almost no legitimate use outside authorized testing |
| 🔴 Critical | T1486 — Ransomware Deployment | `akira_readme.txt` creation is a confirmed IOC |
| 🟠 High | T1003.001 — LSASS Dump | Sysmon EID 10 on LSASS from unknown process is reliable and low noise |
| 🟠 High | T1572 — Ngrok Tunneling | DNS resolution of `ngrok.io` from internal hosts has very limited legitimate use |
| 🟠 High | T1021.001 — RDP to DC | Workstation → domain controller via RDP (Logon Type 10) is rarely legitimate |
| 🟡 Medium | T1087.002 — AD Enumeration | SharpHound/AdFind execution is a reliable pre-lateral-movement indicator |
| 🟡 Medium | T1136 — Account Creation | EID 4720 → 4732 sequence is a reliable persistence signal |
| 🟡 Medium | T1560.001 — Data Staging | Large archive creation in temp paths warrants investigation |

## Key Correlation Chains for Threat Hunting

These multi-event chains represent stronger detection confidence than any single indicator:

**Chain 1 — Pre-encryption kill chain:**
> VSS deletion (T1490) → Service stop (T1489) → File permission change (T1222) → Encryptor execution (T1486)

**Chain 2 — Credential abuse leading to lateral movement:**
> LSASS dump (T1003.001) → Kerberoasting (T1558.003) → RDP to DC (T1021.001) → New account created (T1136)

**Chain 3 — Exfiltration staging:**
> AD enumeration (T1087) → WinRAR archives on shares (T1560.001) → rclone/FileZilla exfil (T1048)

**Chain 4 — Hypervisor attack path:**
> VPN exploit / IAB access (T1190 / T1078) → SSH to hypervisor (T1021.004) → Bulk VM shutdown (T1529) → ESXi/VMDK encryption (T1486)
